# Blog Post Blueprint

## Goal

Establish thought leadership, capture informational search intent, and link
readers to relevant service pages.

## Recommended sections

1. Hero with title, publish date, author name
2. Author bio / E-E-A-T signals (credentials, role, experience)
3. Table of contents (for posts > 1500 words)
4. Introduction — hook the reader, state the problem, preview the value
5. Body sections with H2/H3 structure, data, and examples
6. Key takeaways / summary box
7. Related posts (2-3 internal links)
8. CTA section — link to relevant service or contact page
9. FAQ (optional, 2-3 questions for schema markup)

## Inputs required

- brand strategy (tone of voice)
- SEO brief (keyword targets)
- personas (audience context)
- blog strategy (categories, clusters)

## Internal linking

- Link to 2-3 relevant service pages within body content
- Link to related blog posts in the related posts section
- CTA links to contact or service page
- Pillar page links back to this post (update pillar after publishing)

## SEO requirements

- H1 contains primary keyword naturally
- Meta description includes primary keyword + value proposition
- OG tags with article type, published date, author
- JSON-LD Article schema with author, datePublished, headline
- FAQ schema if FAQ section is present

## Accessibility

- H1 for title, H2 for sections, H3 for subsections
- Images have descriptive alt text
- Links have descriptive anchor text (not "click here")
- Code blocks use proper semantic markup
