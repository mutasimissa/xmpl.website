# Content Quality

- Avoid fluff, filler, and unsupported superlatives
- Every claim must be supportable with evidence — flag gaps for the user
- Write for the real buyer: specific role, real problems, practical language
- Keep CTA clear, consistent, and outcome-oriented across all pages
- Lead with outcomes and benefits, not features
- Address the buyer's problem before presenting the solution
- Use scannable structure: short paragraphs, bullet points, clear headings
- Match tone to `business/02-brand-strategy.md` — every sentence should sound
  like this brand
- Include proof near key claims (credentials, process, results, references)
- Never invent business facts — all content must trace to `business/` files
